Semester 1 Study Plan (Fabric, Python, KQL)

This 14-week plan assumes roughly 10–12 hours per week, focusing on Microsoft Fabric fundamentals, lakehouse engineering with PySpark, Python for data work, and KQL for Real-Time Analytics.

Week Main Theme Fabric Tasks Coding / KQL Tasks Optional AI Tasks Est. Hours
1 Ramp-up & setup - Set up Fabric workspace and explore workloads
- Work through high-level Fabric overview modules 		- Refresh Python basics (syntax, control flow, functions)
- Set up Jupyter / notebooks environment 		- Skim a short ML overview (types of ML, basic terminology) 10–12
2 Workspaces & OneLake - Deepen understanding of workspaces, capacities, items
- Create sample lakehouse, warehouse, and semantic model 		- Start pandas: DataFrames, reading CSVs, simple aggregations - No AI required (buffer week) 10–12
3 Medallion basics - Study medallion architecture (bronze/silver/gold)
- Implement a simple medallion layout in a lakehouse using a small public CSV 		- Continue pandas: joins, groupby, simple charts - Map chosen dataset to possible ML tasks (classification/regression) 10–12
4 Lakehouse creation - Create a dedicated lakehouse for your main public dataset
- Ingest raw data into bronze tables 		- Build an EDA notebook in Python for the chosen dataset (outside Fabric if easier) - Optional: note data quality issues with an eye toward ML later 10–12
5 PySpark & Delta basics - Use Fabric notebooks to read bronze data with PySpark
- Clean and transform data; write to Delta silver tables 		- Start KQL fundamentals: filtering, summarizing, projecting in a small KQL database - Optional: review how logs/time-series data differ from batch tabular data 10–12
6 Transformations & gold layer - Implement richer PySpark transformations (joins, windows)
- Build gold tables (e.g., daily or user-level aggregates) 		- Extend KQL with time-series queries and simple monitoring-style queries - Optional: read about evaluation metrics (accuracy, precision/recall) 10–12
7 Streaming concepts - Learn structured streaming basics in Fabric
- Implement a simulated streaming pipeline (micro-batch from folder or event source) 		- Build KQL queries and simple dashboards for a log or event dataset - Mid-term self-assessment: identify weak areas 10–12
8 Mini-project 1 - Build an end-to-end medallion pipeline for the main public dataset
- Bronze → Silver → Gold in Fabric 		- Wire a simple semantic/reporting layer or queries against gold tables - Document the project (architecture, tables, transformations) 10–12
9 DP-600/700 alignment - Read DP-600 and DP-700 study guides; map your current skills to exam objectives
- Identify gaps in Fabric workloads or patterns 		- Deepen Python data work: more complex analysis and visualizations - Optional: prototype a tiny ML model in a notebook 10–12
10 Fill Fabric gaps - Implement 1–2 missing patterns (e.g., Dataflows Gen2 ingestion, alternative sources)
- Improve performance/structure in existing pipelines 		- Extend KQL work: create a small set of reusable operational queries - Optional: refine ML notebook from Week 9 10–12
11 Mini-project 2 (real-time) - Build a small real-time analytics scenario in Fabric Real-Time Intelligence
- Ingest simulated events into a KQL database 		- Author KQL-based dashboards showing key metrics and trends - Reflect on batch vs real-time trade-offs 10–12
12 Polish & documentation - Refactor notebooks and pipelines for both mini-projects
- Improve naming, configuration, and folder structures 		- Ensure KQL queries and Python notebooks are clean and well-commented - Write 2–3 STAR-style interview stories based on your projects 10–12
13 Capstone assembly - Consolidate all work into a single Fabric workspace and GitHub repo
- Create a clear architecture diagram and README 		- Verify that all code runs cleanly end-to-end - Light DP-600/700 preview; note which objectives remain for next semester 10–12
14 Retrospective & buffer - Write a short retrospective on what you learned and what to change in Semester 2 - Catch up on any incomplete modules or labs - Identify one Kaggle competition or Azure/Fabric hackathon to target next semester 10–12
